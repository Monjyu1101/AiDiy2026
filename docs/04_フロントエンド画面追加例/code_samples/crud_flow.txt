■ M商品 CRUD画面の基本フロー

[カテゴリメニュー] → [一覧画面] → [編集画面]
                         ↑
                   商品IDリンククリック

■ 一覧画面の役割
- /apps/V商品/一覧 でデータ取得
- qTublerFrame + フィルタ/ソート/ページング表示
- 商品IDリンク押下で編集画面へ遷移
- 新規ボタンで新規モードへ遷移

■ 編集画面の役割
- 新規モード: /apps/M商品/登録
- 編集モード: /apps/M商品/取得 → /apps/M商品/変更
- 削除: /apps/M商品/削除（確認ダイアログあり）
- キャンセルで一覧へ戻る

■ M商品でのAPI呼び出し
一覧取得: POST /apps/V商品/一覧
1件取得: POST /apps/M商品/取得
登録:    POST /apps/M商品/登録
変更:    POST /apps/M商品/変更
削除:    POST /apps/M商品/削除
