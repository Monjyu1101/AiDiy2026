■ CRUD画面の基本フロー

[カテゴリメニュー] → [一覧画面] → [編集画面]
                         ↓
                    ダブルクリック
                         ↓
                    [編集画面]

■ 一覧画面の役割
- APIから一覧データ取得
- qTublerテーブルで表示
- 行ダブルクリックで編集画面へ遷移
- 新規作成ボタン

■ 編集画面の役割
- 新規モード: IDなしでアクセス → 登録API
- 編集モード: ID付きでアクセス → 変更API
- 削除ボタン（編集モードのみ）
- キャンセルで一覧へ戻る

■ API呼び出しパターン
一覧取得: POST /core/<テーブル>/一覧
1件取得: POST /core/<テーブル>/取得
登録:    POST /core/<テーブル>/登録
変更:    POST /core/<テーブル>/変更
削除:    POST /core/<テーブル>/削除
