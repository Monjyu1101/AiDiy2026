目的

このドキュメントの目的

AiDiy プロジェクトで M系（マスタ）CRUD 画面を新規追加する手順を解説する。 M車両の実装（frontend_web/src/components/Mマスタ/M車両/）を参照例として、 ディレクトリ作成から一覧・編集画面の実装、ルーティング登録、動作確認までの全工程を扱う。

T系（トランザクション）画面の追加も基本的に同じパターンで実装できる。 その場合はパスプレフィックスを /Mマスタ/M<テーブル名> から /Tトラン/T<テーブル名> に読み替えること。

前提： バックエンドの M系 API（登録・変更・削除・取得・一覧）が実装済みであること。 バックエンド実装の手順は backend_server/AGENTS.md を参照。

実装の流れ

実装ステップ（推奨順序）

  1. ディレクトリ作成 — components/Mマスタ/M<テーブル名>/ と components/Mマスタ/M<テーブル名>/components/ を作成する。 詳細 → 01 ディレクトリ構成
  2. 一覧テーブルコンポーネント作成 — M<テーブル名>一覧テーブル.vue（components/ 配下）を作成する。 POST /apps/V<テーブル名>/一覧 でデータ取得、件数制限・無効も表示 props を実装する。 defineExpose({ loadData }) で親から再読み込みできるようにする。 詳細 → 02 一覧画面
  3. 一覧画面作成 — M<テーブル名>一覧.vue を作成する。 件数制限・無効も表示のチェックボックス UI、新規ボタン、再検索ボタンを実装する。 詳細 → 02 一覧画面
  4. 編集画面作成 — M<テーブル名>編集.vue を作成する。 モード判定（新規/編集/表示）、フォーム reactive、バリデーション、論理削除（qConfirm）を実装する。 詳細 → 03 編集画面
  5. ルーティング登録 — appsRouter.ts に一覧・編集の2ルートを追加する。 詳細 → 04 ルートとメニュー
  6. メニュー追加 — Mマスタ.vue にメニューカードを追加する。 詳細 → 04 ルートとメニュー
  7. 動作確認 — 一覧表示・新規登録・変更・削除・件数制限・無効も表示を実際に操作して確認する。 詳細 → 05 チェックリスト
