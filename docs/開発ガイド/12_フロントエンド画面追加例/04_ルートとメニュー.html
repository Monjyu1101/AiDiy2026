1. ルートパスの命名規則

M系マスタ画面のルートパスは以下の規則に従う。 テーブル名は日本語のまま使用し、ファイルパスと完全に対応させる。

種別パスパターン例
M系 一覧/Mマスタ/M<テーブル名>/一覧/Mマスタ/M車両/一覧
M系 編集/Mマスタ/M<テーブル名>/編集/Mマスタ/M車両/編集
T系 一覧/Tトラン/T<テーブル名>/一覧/Tトラン/T配車/一覧
T系 編集/Tトラン/T<テーブル名>/編集/Tトラン/T配車/編集

ルート name はパスの区切りを除いた日本語（例: 'M車両一覧', 'M車両編集'）。 全ての業務画面に meta: { requiresAuth: true, title: '...' } を設定する。

2. appsRouter.ts へのルート追加

frontend_web/src/router/appsRouter.ts の M系ブロック（// ---- M系 (マスタ) ----）に 一覧・編集の2ルートを追加する。既存の M商品・M車両の直後が推奨配置。 

 ...

ポイント：

  • M系は /Mマスタ/... 配下に配置する
  • 一覧と編集の2ルートをセットで登録する
  • meta.requiresAuth は必ず true にする（認証ガード）
  • meta.title は画面名と揃える（ブラウザタブ表示）
  • component は動的 import（() => import('...')）を使う
// appsRouter.ts 追加例（M<テーブル名> の場合）
{
    path: '/Mマスタ/M<テーブル名>/一覧',
    name: 'M<テーブル名>一覧',
    component: () => import('../components/Mマスタ/M<テーブル名>/M<テーブル名>一覧.vue'),
    meta: { requiresAuth: true, title: 'M<テーブル名>一覧' }
},
{
    path: '/Mマスタ/M<テーブル名>/編集',
    name: 'M<テーブル名>編集',
    component: () => import('../components/Mマスタ/M<テーブル名>/M<テーブル名>編集.vue'),
    meta: { requiresAuth: true, title: 'M<テーブル名>編集' }
},

3. meta: { requiresAuth, title } の設定

meta.requiresAuth: true を設定すると router/index.ts の ナビゲーションガードが認証チェックを行い、未ログインの場合はログイン画面へリダイレクトする。 meta.title はブラウザのタブタイトルに使われる。 

// router/index.ts のナビゲーションガード（既存コード）
router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isLoggedIn()) {
    next('/login');
  } else {
    document.title = String(to.meta.title || 'AiDiy');
    next();
  }
});

全業務画面（M系・T系・V系・S系）は必ず requiresAuth: true を設定する。 ログイン画面・エラーページのみ false または省略。

4. Mマスタ.vue へのメニューカード追加

frontend_web/src/components/Mマスタ.vue にメニューカードを追加する。 既存の M商品・M車両カードのすぐ後に同じ形式で追記する。 

 ...

新しいテーブル用のカードを追加する場合はこのパターンをコピーして M商品 を M<テーブル名> に置き換える。 

<!-- Mマスタ.vue に追加するカード（M<テーブル名> 用） -->
<div class="menu-card">
  <div class="menu-card-title">
    <span class="icon">M名</span>
    M<テーブル名>
  </div>
  <div class="menu-card-description">
    <テーブルの説明文・ビジネスドメインに合わせて記載>
  </div>
  <router-link to="/Mマスタ/M<テーブル名>/一覧" class="menu-card-link">
    開く →
  </router-link>
</div>

注意： Vue コンポーネントのタグ名は ASCII のみ有効。 日本語ファイル名のコンポーネントを動的に使う場合は <component :is="コンポーネント変数" /> 形式を使う。 <M車両一覧 /> のような日本語タグは無効。
