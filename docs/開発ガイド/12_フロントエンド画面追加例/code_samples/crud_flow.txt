■ M商品 CRUD画面の基本フロー

[カテゴリメニュー] → [一覧画面] → [編集画面]
 ↑
 商品IDリンククリック

■ 一覧画面の役割
- /apps/M商品分類/一覧 で商品分類プルダウンを取得
- /apps/V商品/一覧 でデータ取得（商品分類ID / 件数制限 / 無効も表示を送る）
- qTublerFrame + フィルタ/ソート/ページング表示
- 商品ID / 商品名リンク押下で編集画面へ遷移
- 新規ボタンで新規モードへ遷移

■ 編集画面の役割
- 新規モード: /apps/M商品/登録
- 編集モード: /apps/M商品/取得 → /apps/M商品/変更
- 削除: /apps/M商品/削除（確認ダイアログあり）
- キャンセルで一覧へ戻る

■ M商品でのAPI呼び出し
商品分類一覧: POST /apps/M商品分類/一覧
一覧取得: POST /apps/V商品/一覧
1件取得: POST /apps/M商品/取得
登録: POST /apps/M商品/登録
変更: POST /apps/M商品/変更
削除: POST /apps/M商品/削除（有効=false）
