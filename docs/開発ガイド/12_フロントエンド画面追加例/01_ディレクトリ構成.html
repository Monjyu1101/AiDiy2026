CRUD 画面のフォルダ構造

M系マスタ画面は 3ファイル構成 で実装します。M車両 を例に示します。

frontend_web/src/components/Mマスタ/M<テーブル名>/
├── M<テーブル名>一覧.vue            # 一覧ページ（件数制限・無効も表示チェックボックス）
├── M<テーブル名>編集.vue            # 編集/新規作成ページ（登録・変更・削除）
└── components/
    └── M<テーブル名>一覧テーブル.vue # qTublerFrame + V系API呼び出し

新規追加時のフォルダ作成コマンド:

# Windows
mkdir frontend_web\src\components\Mマスタ\M新テーブル
mkdir frontend_web\src\components\Mマスタ\M新テーブル\components

# Linux/Mac
mkdir -p frontend_web/src/components/Mマスタ/M新テーブル/components

各ファイルの役割

ファイル 役割 主な内容
M<テーブル名>一覧.vue 一覧ページ（親） 件数制限・無効も表示チェックボックス、再検索ボタン、新規ボタン、 <component :is="..."> で子コンポーネントを呼び出し、 handleReload() で子の loadData() を呼ぶ
M<テーブル名>編集.vue 編集/新規作成ページ 1件取得・登録・変更・削除の API 呼び出し、 フォーム入力バリデーション、 完了後に 戻URL へリダイレクト
components/M<テーブル名>一覧テーブル.vue 一覧テーブル（子） V系API (/apps/V<テーブル名>/一覧) でデータ取得、 qTublerFrame でテーブル表示、 件数制限 / 無効も表示 / 有効列表示 / 戻URL を props で受け取る、 defineExpose({ loadData }) で親から呼び出し可能

コンポーネント間の連携

親（一覧.vue）から子（一覧テーブル.vue）へ 4つの props を渡します。

prop 名 型 デフォルト 説明
件数制限 Boolean true true: 上限1000件取得 / false: 全件取得
無効も表示 Boolean false false: 有効レコードのみ / true: 無効レコードも含む
有効列表示 Boolean false 「有効」列をテーブルに表示するか（通常は 無効も表示 と連動）
戻URL String '' 行クリック時に編集画面へ渡す戻URL（編集完了後にここへ戻る）

親での宣言パターン（<script setup> 内）:

const 件数制限 = ref(true);
const 無効も表示 = ref(false);
const 有効列表示 = computed(() => 無効も表示.value); // 無効も表示ONのとき「有効」列を表示

子へ渡すテンプレートパターン:

<!-- ※ Vue コンポーネントタグ名は ASCII のみ有効。日本語コンポーネントは <component :is="..."> で使用する -->
<component
  :is="テーブル名一覧テーブル"
  ref="テーブル名一覧テーブルRef"
  :件数制限="件数制限"
  :無効も表示="無効も表示"
  :有効列表示="有効列表示"
  :戻URL="編集戻URL"
/>

M車両の具体例（ファイルパス）

実際の実装ファイルパス:

ファイル パス
一覧ページ frontend_web/src/components/Mマスタ/M車両/M車両一覧.vue
編集ページ frontend_web/src/components/Mマスタ/M車両/M車両編集.vue
一覧テーブル frontend_web/src/components/Mマスタ/M車両/components/M車両一覧テーブル.vue

M車両は 件数制限 / 無効も表示 / 有効列表示 / 戻URL の標準パターンを全て実装しており、新規テーブル追加時のテンプレートとして最も参照しやすい実装例です。
