1. 編集画面の役割

M<テーブル名>編集.vue は登録・変更・論理削除の3操作を1画面で担う。 クエリパラメータ モード で画面の動作モードを切り替える。

操作API エンドポイント説明
登録POST /apps/M<テーブル名>/登録新規レコードを作成する
変更POST /apps/M<テーブル名>/変更既存レコードを更新する
削除（論理削除）POST /apps/M<テーブル名>/削除有効フラグを False にセットする

2. モード判定（クエリパラメータ）

ルートのクエリパラメータ モード の値で動作モードを決定する。 モード=新規 の場合は新規作成、モード=表示 かつ ID パラメータあり の場合は閲覧専用、 ID パラメータのみの場合は編集モードになる。 

const applyQueryParams = async (query) => {
  message.value = '';
  resetValidation();

  if (query.モード === '新規') {
    mode.value = 'create';
    detailData.value = null;
    resetForm();
    return;
  }

  if (query.モード === '表示' && query.車両ID) {
    mode.value = 'view';
    await loadDetail(query.車両ID);
    return;
  }

  if (query.車両ID) {
    mode.value = 'edit';
    await loadDetail(query.車両ID);
    return;
  }

  mode.value = 'create';
  detailData.value = null;
  resetForm();
};

// onMounted と watch(route.query) の両方から呼ぶ
onMounted(async () => { await applyQueryParams(route.query); });
watch(() => route.query, async (query) => { await applyQueryParams(query); });
クエリ例mode 値説明
?モード=新規createフォーム空で登録モード
?車両ID=V001edit既存データを取得して変更モード
?モード=表示&車両ID=V001view読み取り専用（全入力欄 readonly）

isCreateMode / isEditMode / isViewMode は computed() で定義しておき、 テンプレートの :readonly や :disabled に使う。 

const isCreateMode = computed(() => mode.value === 'create');
const isEditMode   = computed(() => mode.value === 'edit');
const isViewMode   = computed(() => mode.value === 'view');

3. フォーム reactive オブジェクトの定義

reactive() でフォームデータ・エラー・touched の3オブジェクトを定義する。 有効フラグ（有効: true）を必ず含めること。 論理削除はバックエンドがこのフラグを False にセットする。 

const form = reactive({
  車両ID: '',
  車両名: '',
  車両備考: '',
  有効: true   // 論理削除フラグ。false = 無効（論理削除済み）
});

const errors = reactive({
  車両ID: '',
  車両名: '',
  車両備考: ''
});

const touched = reactive({
  車両ID: false,
  車両名: false,
  車両備考: false
});

// 必須フィールドリスト（バリデーション対象）
const requiredFields = computed(() => ['車両ID', '車両名']);

// APIレスポンスをフォームに適用するヘルパー
const applyDataToForm = (data) => {
  form.車両ID    = data?.車両ID    || '';
  form.車両名    = data?.車両名    || '';
  form.車両備考  = data?.車両備考  || '';
  form.有効      = data?.有効      ?? true;
};

4. API 呼び出しパターン（取得・登録・変更）

全 API は POST メソッド。レスポンスは { status: "OK"/"NG", message: "...", data: {...} } 形式。 登録・変更はモードに応じてエンドポイントを切り替える。 

// 1件取得
const loadDetail = async (vehicleId) => {
  try {
    const res = await apiClient.post('/apps/M車両/取得', { 車両ID: String(vehicleId) });
    if (res.data.status === 'OK' && res.data.data) {
      detailData.value = res.data.data;
      applyDataToForm(res.data.data);
    } else {
      void qMessage(res.data.message || '車両情報の取得に失敗しました。', 'error');
    }
  } catch (e) {
    void qMessage('車両情報の取得でエラーが発生しました。', 'error');
  }
};

// 登録・変更（モードで自動切替）
const saveData = async () => {
  if (!validateForm()) {
    void qMessage('入力内容を確認してください。', 'error');
    return;
  }
  try {
    const endpoint = isCreateMode.value ? '/apps/M車両/登録' : '/apps/M車両/変更';
    const res = await apiClient.post(endpoint, {
      車両ID:   form.車両ID,
      車両名:   form.車両名,
      車両備考: form.車両備考,
      有効:     form.有効
    });
    if (res.data.status === 'OK') {
      handleSuccess(res.data.message);
    } else {
      void qMessage(res.data.message || '保存に失敗しました。', 'error');
    }
  } catch (e) {
    void qMessage('保存に失敗しました。', 'error');
  }
};

5. 論理削除の実装パターン

削除は物理削除ではなく論理削除（バックエンドが 有効=False にセット）。 APIレスポンスの message は "M車両「V001」の有効をオフにしました" 形式になる。 削除前に qConfirm で確認ダイアログを表示すること（native confirm は使用禁止）。 

import { qConfirm, qMessage } from '../../../utils/qAlert';

const deleteData = async () => {
  if (!form.車両ID) return;

  // qConfirm で確認（native confirm は禁止）
  const confirmed = await qConfirm(
    `M車両「${form.車両ID}」を削除しますか？この操作は取り消せません。`
  );
  if (!confirmed) return;

  try {
    const res = await apiClient.post('/apps/M車両/削除', { 車両ID: form.車両ID });
    if (res.data.status === 'OK') {
      // res.data.message = "M車両「V001」の有効をオフにしました"
      handleSuccess(res.data.message);
    } else {
      void qMessage(res.data.message || '削除に失敗しました。', 'error');
    }
  } catch (e) {
    void qMessage('削除に失敗しました。', 'error');
  }
};

削除後の一覧画面では「無効も表示」チェックボックスをオンにすることで、 論理削除済みレコード（有効=False）も確認できる。

6. バリデーション（必須チェック・エラー表示）

validateField() で1フィールドずつ、validateForm() で全フィールドをまとめてチェックする。 touched フラグで「送信前の未入力には警告を出さない」制御を行う。 

const validateField = (field, showMsg = true) => {
  const value = String(form[field] ?? '').trim();
  if (!requiredFields.value.includes(field)) {
    errors[field] = ''; return true;
  }
  if (!value) {
    // showMsg=false のときは 'ERROR' を入れて input-error クラスだけ付ける
    errors[field] = showMsg ? `${field}は必須です。` : 'ERROR';
    return false;
  }
  errors[field] = '';
  return true;
};

// blur 時: touched を立てて個別バリデーション
const handleBlur  = (field) => { touched[field] = true; validateField(field); };
// input 時: touched が立っている時だけ再バリデーション
const handleInput = (field) => { if (touched[field]) validateField(field); };

// 保存前: 全フィールドを一括チェック
const validateForm = () => {
  let isValid = true;
  requiredFields.value.forEach((field) => {
    touched[field] = true;
    if (!validateField(field, false)) isValid = false;
  });
  return isValid;
};

テンプレート側では :class="{ 'input-error': errors.車両ID }" で赤枠を表示し、 v-if="errors.車両ID && errors.車両ID !== 'ERROR'" でエラーメッセージを表示する。 

<input
  id="form-vehicle-id"
  type="text"
  v-model="form.車両ID"
  :readonly="!isCreateMode || isViewMode"
  :class="{ 'input-error': errors.車両ID }"
  @blur="handleBlur('車両ID')"
  @input="handleInput('車両ID')"
/>
<span v-if="errors.車両ID" class="input-alert">!</span>
<div v-if="errors.車両ID && errors.車両ID !== 'ERROR'" class="field-error">
  {{ errors.車両ID }}
</div>

7. 登録・変更・削除成功後のリダイレクト

登録・変更・削除の成功後は一覧画面へリダイレクトし、 message と type=success をクエリパラメータで渡す。 一覧画面側がそれを受け取って成功メッセージを表示する。 クエリパラメータ 戻URL が指定されている場合はそちらへ遷移する。 

const handleSuccess = (messageText) => {
  if (戻URL.value) {
    // 戻URLが指定されている場合（全角→半角変換を挟む）
    router.push(toHalfwidthUrl(戻URL.value));
    return;
  }
  // 一覧画面へリダイレクト（成功メッセージ付き）
  router.push({
    path: '/Mマスタ/M車両/一覧',
    query: { message: messageText, type: 'success' }
  });
};

一覧画面では route.query.message を受け取り qMessage() で表示する。 type=success / type=error によって表示スタイルが変わる。
