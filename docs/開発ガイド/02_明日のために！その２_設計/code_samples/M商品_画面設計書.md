# M商品 画面設計書

**更新日**: 2026-04-04  
**対象画面**: `M商品一覧`, `M商品編集`

---

## 1. 画面一覧

| 画面名 | URL | 用途 |
|---|---|---|
| M商品一覧 | `/Mマスタ/M商品/一覧` | 一覧、絞り込み、遷移 |
| M商品編集 | `/Mマスタ/M商品/編集` | 新規、編集、表示 |

---

## 2. M商品一覧

### 2.1 概要

商品一覧画面です。実データは `V商品/一覧` から取得します。

### 2.2 主な UI

- タイトル `【 M商品 】`
- 戻るボタン
  - `戻URL` が query にある場合のみ表示
- 商品分類プルダウン
- `再検索`
- `新規`
- `件数制限` チェック
- `無効も表示` チェック
- qTubler 一覧テーブル

### 2.3 一覧列

| 列 | 説明 |
|---|---|
| 商品ID | リンク。クリックで編集画面へ |
| 商品名 | リンク。クリックで編集画面へ |
| 単位 | 表示 |
| 商品分類名 | 表示 |
| 商品備考 | 表示 |
| 有効 | チェック表示 |
| 更新日時 | 表示 |
| 更新利用者名 | 表示 |

### 2.4 一覧取得 API

```json
POST /apps/V商品/一覧
{
  "商品分類ID": "...",
  "件数制限": true,
  "無効も表示": false
}
```

### 2.5 遷移

#### 新規

`新規` 押下時:

```text
/Mマスタ/M商品/編集?モード=新規&戻URL=<現在一覧URL>
```

#### 編集

商品ID または 商品名クリック時:

```text
/Mマスタ/M商品/編集?モード=編集&商品ID=<商品ID>&戻URL=<現在一覧URL>
```

## 3. M商品編集

### 3.1 モード

- 新規
  - query: `モード=新規`
- 編集
  - query: `商品ID=...`
- 表示
  - query: `モード=表示&商品ID=...`

### 3.2 タブ構成

- `内容`
- `その他`

### 3.3 内容タブ

| 項目 | 入力 | 必須 | 備考 |
|---|---|---|---|
| 商品ID | text | ○ | 新規時のみ編集可 |
| 商品名 | text | ○ | |
| 単位 | text | ○ | |
| 商品分類ID | select | ○ | M商品分類一覧から選択 |
| 商品備考 | textarea | - | |

### 3.4 その他タブ

| 項目 | 表示内容 |
|---|---|
| 有効 | チェックボックス |
| 登録日時 | read only |
| 登録利用者 | read only |
| 登録端末 | read only |
| 更新日時 | read only |
| 更新利用者 | read only |
| 更新端末 | read only |

### 3.5 ボタン

- `一覧に戻る`
- `登録` または `更新`
- `削除`
  - 編集モードのみ

### 3.6 バリデーション

- 商品ID 必須
- 商品名 必須
- 単位 必須
- 商品分類ID 必須

画面上は未入力時に各項目へエラー表示を出します。

## 4. API 連携

### 4.1 初期表示

新規:

- `M商品分類/一覧` を読み込み
- フォーム初期化

編集 / 表示:

1. `M商品分類/一覧`
2. `M商品/取得`

### 4.2 保存

新規:

```json
POST /apps/M商品/登録
```

編集:

```json
POST /apps/M商品/変更
```

### 4.3 削除

```json
POST /apps/M商品/削除
```

注意:

- 削除は物理削除ではなく `有効=false` への更新です。

---

## 5. 画面遷移

```text
M商品一覧
  ├─ 新規 → M商品編集(新規)
  └─ 商品ID/商品名クリック → M商品編集(編集)

M商品編集
  ├─ 登録/更新成功 → 戻URL があればそこへ戻る
  └─ 戻URL がなければ /Mマスタ/M商品/一覧 へ戻る
```
