# M商品 API設計書

**更新日**: 2026-04-23  
**対象 API**: `M商品`, `V商品`

---

## 1. 対象エンドポイント

| エンドポイント | メソッド | 用途 |
|---|---|---|
| `/apps/V商品/一覧` | POST | 一覧取得 |
| `/apps/M商品/一覧` | POST | M商品テーブルの単純一覧 |
| `/apps/M商品/取得` | POST | 1件取得 |
| `/apps/M商品/登録` | POST | 新規登録 |
| `/apps/M商品/変更` | POST | 更新 |
| `/apps/M商品/削除` | POST | **有効フラグ OFF** |

補足:

- 画面一覧は通常 `V商品/一覧` を使います。
- `M商品/削除` は物理削除ではありません。

---

## 2. 共通仕様

### 認証

- JWT Bearer Token
- 有効期限: **60分**
- フロントエンド Web の C/M/T 操作系APIでは送信前に `/core/auth/トークン更新` による延長を試みます
- 認証不要なのは `/core/auth/ログイン` など一部のみ

### 統一レスポンス

```json
{
  "status": "OK",
  "message": "メッセージ",
  "data": {}
}
```

エラー時:

```json
{
  "status": "NG",
  "message": "メッセージ",
  "error": {
    "code": "NOT_FOUND"
  }
}
```

## 3. データ項目

### M商品

| 項目 | 型 | 説明 |
|---|---|---|
| 商品ID | string | 主キー |
| 商品名 | string | 商品名 |
| 単位 | string | 単位 |
| 商品分類ID | string | M商品分類 を参照 |
| 商品備考 | string \| null | 備考 |

### V商品 追加項目

| 項目 | 型 | 説明 |
|---|---|---|
| 商品分類名 | string \| null | JOIN された分類名 |

## 4. 一覧取得

### 4.1 `POST /apps/V商品/一覧`

#### リクエスト例

```json
{
  "商品分類ID": "A",
  "件数制限": true,
  "無効も表示": false
}
```

#### レスポンス例

```json
{
  "status": "OK",
  "message": "V商品一覧を取得しました",
  "data": {
    "items": [
      {
        "商品ID": "A001",
        "商品名": "牛飼料 #001",
        "単位": "Kg",
        "商品分類ID": "A",
        "商品分類名": "完成品",
        "商品備考": "",
        "有効": true,
        "更新日時": "2026-04-04 20:00:00",
        "更新利用者名": "system"
      }
    ],
    "total": 1,
    "limit": 1000
  }
}
```

補足:

- `件数制限=true` のとき `limit` は `1000`
- `件数制限=false` のとき `limit` は `null`

### 4.2 `POST /apps/M商品/一覧`

内部向けの単純一覧です。画面一覧では通常使いません。

## 6. 登録

### `POST /apps/M商品/登録`

#### リクエスト

```json
{
  "商品ID": "A999",
  "商品名": "新商品",
  "単位": "Kg",
  "商品分類ID": "A",
  "商品備考": "テスト登録",
  "有効": true
}
```

#### 成功時

- `status: OK`
- `message: "商品を作成しました"`

#### 主なエラー

| 条件 | code | message |
|---|---|---|
| 商品ID重複 | `DUPLICATE` | この商品IDは既に登録されています |

---

## 7. 変更

### `POST /apps/M商品/変更`

#### リクエスト

```json
{
  "商品ID": "A001",
  "商品名": "牛飼料 改",
  "単位": "Kg",
  "商品分類ID": "A",
  "商品備考": "更新",
  "有効": true
}
```

#### 成功時

- `status: OK`
- `message: "商品を更新しました"`

#### 主なエラー

| 条件 | code | message |
|---|---|---|
| 対象なし | `NOT_FOUND` | 指定された商品が見つかりません |

---

## 8. 削除

### `POST /apps/M商品/削除`

#### リクエスト

```json
{
  "商品ID": "A001"
}
```

#### 挙動

- 対象商品の `有効` を `false` に更新
- 監査項目も更新
- 物理削除はしない

#### 成功時

- `status: OK`
- `message: "商品の有効をオフにしました"`

---

## 9. フロント連携メモ

- 一覧画面: `frontend_web/src/components/Mマスタ/M商品/M商品一覧.vue`
- 一覧データ: `frontend_web/src/components/Mマスタ/M商品/components/M商品一覧テーブル.vue`
- 編集画面: `frontend_web/src/components/Mマスタ/M商品/M商品編集.vue`

フロント側では主に次を送ります。

- `商品分類ID`
- `件数制限`
- `無効も表示`
- 編集時の query:
  - `モード=新規`
  - `商品ID=...`
  - `戻URL=...`
