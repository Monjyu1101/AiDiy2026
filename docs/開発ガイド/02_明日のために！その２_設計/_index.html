02_明日のために！その２_設計

この章の目的

この章は、M商品を題材に「設計書を先に固めてから実装する」ための実践ガイドです。対象プロジェクトは workspace/AiDiy2026 です。

  • 外部設計（画面）と内部設計（API・業務ロジック）を分離して整理できる
  • DB設計まで落とし込んだうえで実装着手できる
  • 設計成果を 03（バックエンド）と 04（フロントエンド）へ接続できる

設計と実装の順序

フェーズ推奨順序理由
設計外部設計 → 内部設計 → DB設計利用者視点から仕様を固め、内部仕様へ詳細化するため
実装DB → バックエンド → フロントエンド依存関係を下流から解消し、手戻りを減らすため
