02_Schema作成

SchemaはAPI契約の中心です。入力（Create/Update/Get/Delete）と出力（レスポンス）を分け、型と必須項目を明確にします。

作成するSchema

Schema名用途
M商品Base共通入力項目（商品名、単位、商品備考）
M商品Create新規登録用（商品ID含む）
M商品Update更新用（商品ID必須、他はOptional）
M商品Get1件取得用（商品ID）
M商品Delete削除用（商品ID）
M商品レスポンス用（監査項目含む）

共通レスポンス形式

Routerでは ResponseBase を返却します。

{
  "status": "OK | NG",
  "message": "メッセージ",
  "data": {},
  "error": {}
}

共通リクエスト形式：ListRequest

apps_schema/common.py（および core_schema/common.py）に定義されている一覧取得用の共通スキーマです。

from pydantic import BaseModel
from typing import Optional

class ListRequest(BaseModel):
    件数制限: Optional[bool] = True   # True のとき LIMIT 1000 を適用
    無効も表示: Optional[bool] = False  # False のとき 有効=1 のレコードのみ表示

使い分け:

API分類リクエスト上限件数
V系・T系の一覧request: Optional[ListRequest] = Noneデフォルト1,000件（件数制限=False で無制限）
M系・C系の一覧リクエストボディなし固定10,000件（MAX_ITEMS = 10000）

ポイント:

  • request を省略（None）した場合でも、list_controls.py のデフォルト（件数制限あり・有効のみ）が適用される
  • V系の一覧ではJOINを伴う生SQLに条件を組み込むため、append_active_condition / get_limit_clause を使う
  • T系・M系でORMを使う場合は apply_active_filter / get_list_limit を使う

チェック

  • [ ] レスポンスSchemaに監査項目が含まれている
  • [ ] V系・T系の一覧Routerでは ListRequest を Optional で受け取るようになっている
