03_明日のために！その３_バックエンド開発

この章の目的

この章では、M商品を題材に workspace/AiDiy2026/backend_server へバックエンドAPIを追加する手順を学びます。

  • Schema → Model → CRUD → Router の実装順序を理解する
  • AiDiyのPOST中心API設計と共通レスポンス形式に合わせて実装する
  • V系（View系）を生SQLで実装する方針を理解する
  • Swagger UIで動作確認し、不具合の切り分けができる

前提

  • プロジェクトルート: workspace/AiDiy2026
  • バックエンドサーバー: core=8091, apps=8092
  • M商品は apps サーバー（8092）で扱う
  • DBは SQLite（workspace/AiDiy2026/backend_server/_data/AiDiy/database.db）
