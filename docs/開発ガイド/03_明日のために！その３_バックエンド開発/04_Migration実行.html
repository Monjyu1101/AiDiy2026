04_DB再作成と反映

このプロジェクトの標準運用は SQLite + create_all() です。Alembicのようなマイグレーションツールは使いません。

新規テーブル追加時の手順（M商品のようなケース）

M商品 は新規テーブルなので、既存DBを削除しなくても create_all() だけでテーブルが追加されます。

  1. backend_server/apps_models/__init__.py に M商品 のimportを追加する
  2. backend_server/apps_main.py の database.Base.metadata.create_all(bind=database.engine, tables=[...]) のリストへ apps_models.M商品.__table__ を追加する
  3. サーバーを再起動する（python _start.py または個別に uvicorn 起動）
cd workspace/AiDiy2026
python _start.py

注意: create_all() は「まだ存在しないテーブル」だけを作成し、既存テーブルのデータは変更しません。新規テーブル追加のためだけに database.db を削除する必要はありません。

既存テーブルのカラム追加・削除・型変更が必要な場合

create_all() は既存テーブルへのカラム追加・削除・型変更には効きません。この場合は手動SQL（ALTER TABLE やテーブル再作成）が必要です。手順は .aidiy/knowledge/backend_server,スキーマ変更手順.md を参照してください。

開発環境で「既存データを残す必要がない」場合に限り、database.db を削除して全テーブルを作り直す方法も選択肢になります。ただし実データがある環境では行わないでください。

生成確認

  • http://localhost:9098/docs でM商品APIが表示される
  • SQLiteブラウザで M商品 テーブルが作成されている

初期データ投入の考え方

新規M系（例: M○○）を追加した場合、backend_server/apps_crud/init.py の init_db_data() に初期化処理を必ず追加してください。初回確認で一覧が空だと確認品質が落ちるため、可能な限りサンプルデータを投入します。既存データがある場合はスキップし、冪等性を保ちます。
