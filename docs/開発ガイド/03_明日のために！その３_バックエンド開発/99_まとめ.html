99_まとめ

この章では、M商品を通してバックエンド機能追加の一連手順を実施しました。実装の再現性を上げるため、最後に要点を固定します。

実施内容の要約

  1. Schema作成（API契約）
  2. Model作成（DB構造）
  3. SQLite再作成でテーブル反映
  4. CRUD作成（DB操作）
  5. Router作成（認証/レスポンス/業務判定）
  6. View作成（生SQL一覧API）
  7. Swagger UIで動作確認

重要原則

  • POST中心API設計を維持する
  • レスポンス形式は status/message/data/error で統一する
  • 監査項目を全テーブルで統一して扱う
  • V系は CREATE VIEW ではなく Router内生SQLで実装する
  • M系は apps_main.py (8092) 側に登録する
