03_ルーティング設定

Vue Routerへ一覧・編集ルートを追加します。画面遷移を安定させるため、name/path/queryの対応を先に決めます。

設定対象

ルーターはバックエンドと対応する形で分割されています。M系（マスタ）はappsRouter.tsに追加します。

ファイル対象系
src/router/appsRouter.tsM系, T系, V系, S系 ← M商品はここ
src/router/coreRouter.tsC系（C管理, C権限, C利用者, C採番）, A系（AIコア）
src/router/index.tsベースルート（ログイン, X系）+ ルーター本体

追加するルート（例）

ルート名path用途
M商品一覧/Mマスタ/M商品/一覧一覧表示
M商品編集/Mマスタ/M商品/編集新規/編集/表示

クエリパラメータ設計

  • モード=新規: 新規入力
  • 商品ID=ITEM001: 編集対象指定
  • モード=表示 + 商品ID: 参照モード

実装ポイント

  • 画面名とルート名を一致させる
  • 認証が必要な画面は meta.requiresAuth を設定
  • 一覧→編集、編集→一覧の遷移を router.push で統一

AGENTS準拠メモ

  • M/T/V/S系のルート定義は src/router/appsRouter.ts に追加する（index.tsは触らない）
  • C系/A系のルート定義は src/router/coreRouter.ts に追加する
  • コンポーネント読み込みは遅延読み込み（() => import(...)）を基本にする
  • 日本語URLリロード時の復元（decode対応）を壊さないよう既存実装に合わせる

チェック

  • [ ] 直接URLアクセスで一覧/編集が表示される
  • [ ] クエリを変えるとモードが正しく切り替わる
  • [ ] 未ログイン時はガードで遷移制御される
