07_動作確認

一覧と編集のE2E動作を確認します。最低限、登録→更新→削除の一連操作を通してください。

事前条件

  • フロント: http://localhost:8090
  • Apps API: http://localhost:8092/docs
  • ログイン済み（例: admin）

E2Eテスト手順

  1. 一覧画面を開き、既存データ表示を確認
  2. 新規ボタンから編集画面へ移動し、商品を登録
  3. 一覧に戻って対象商品を開き、内容を変更して保存
  4. 再度開いて削除を実行
  5. 一覧から対象が消えることを確認

件数制限・無効も表示の動作確認

  1. 一覧画面を開き、件数制限 チェックボックスをOFF → 全件取得されることを Network タブで確認（件数制限: false がリクエストボディに含まれる）
  2. 件数制限をONに戻し、リクエストボディが 件数制限: true になることを確認
  3. 無効も表示 をONにすると「有効」列がテーブルに追加されることを確認
  4. 無効レコード（有効=false）がテーブルに表示されることを確認
  5. 無効も表示をOFFに戻すと「有効」列が消え、有効レコードのみ表示されることを確認

論理削除（無効化）の動作確認

このシステムの「削除」は物理削除ではなく 論理削除（有効フラグのOFF）です。 フロントエンドでは「有効をオフ」「無効にする」などのボタン名で実装します。

  1. 対象レコードの編集画面を開き、削除（無効化）を実行
  2. 無効も表示=false の状態で一覧に戻ると、対象レコードがリストから消えることを確認
  3. 無効も表示=ON に切り替えると、対象レコードが「有効=false」として一覧に現れることを確認
  4. 再度編集画面を開いて有効を戻せば復元できることを確認

確認観点

観点期待結果
メッセージ表示成功/失敗メッセージが適切に表示される
入力検証必須未入力で保存できない
URL連携モードと商品IDクエリが正しく解釈される
API連携Networkで期待エンドポイントにPOSTされる
件数制限リクエストボディに 件数制限: true/false が含まれる
無効も表示リクエストボディに 無効も表示: true/false が含まれる
有効列無効も表示ONで「有効」列が表示、OFFで非表示になる
論理削除削除後は無効も表示OFFの一覧から消え、ONで有効=falseとして残る

不具合切り分け

  • 画面が出ない: router pathとcomponent importを確認
  • APIエラー: 8092側のRouter/URL/認証を確認
  • 値が反映されない: formバインディングとpayloadを確認
  • 一覧更新されない: 子コンポーネントの再読み込み呼び出しを確認

AGENTS準拠チェック

  • Networkで Authorization: Bearer ... が自動付与される（apiClient interceptor）
  • 401応答時にログアウトへ遷移する（認証フローと整合）
  • 日本語URLをリロードしても対象画面に復帰できる

チェック

  • [ ] 登録/更新/削除を通せる
  • [ ] 主要エラー時にユーザーへ通知できる
  • [ ] 再読み込み後も画面状態が破綻しない
