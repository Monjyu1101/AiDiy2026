05_一覧画面作成

一覧画面はページ全体のレイアウトと操作導線を担当します。実データ表示は子コンポーネント（一覧テーブル）へ委譲します。

対象ファイル

workspace/AiDiy2026/frontend_web/src/components/Mマスタ/M商品/M商品一覧.vue

主な要素

  • 再検索ボタン
  • 新規ボタン（編集画面の新規モードへ遷移）
  • 成功/失敗メッセージ表示
  • 一覧テーブルコンポーネントの埋め込み
  • 件数制限チェックボックス（デフォルト: ON）
  • 無効も表示チェックボックス（デフォルト: OFF）

実装ポイント

  • 一覧再読み込みは子コンポーネントの loadData を呼ぶ
  • 編集画面から戻る際のクエリメッセージを表示し、表示後にURLを整理する
  • ページコンテナとツールバーは一貫したUIに揃える

件数制限・無効も表示パターン（実装必須）

一覧画面では 件数制限 と 無効も表示 の2つのチェックボックスを設け、 それぞれを ref で管理して一覧テーブルコンポーネントへ props として渡します。 実際の実装は C利用者一覧.vue を参照してください。

動作仕様

フラグデフォルト動作
件数制限ON（true）上限1000件で取得（通常運用）
件数制限OFF（false）全件取得（V系・T系のみ対応）
無効も表示OFF（false）有効=true のレコードのみ表示
無効も表示ON（true）無効レコードも含めて表示。テーブルに「有効」列が自動追加される

実装コード（C利用者一覧.vue より抜粋）

<script setup lang="ts">
import { ref, computed } from 'vue';
import UserListTable from './components/C利用者一覧テーブル.vue';

const userListTableRef = ref(null);
const 件数制限 = ref(true);      // デフォルトON
const 無効も表示 = ref(false);   // デフォルトOFF
const 有効列表示 = computed(() => 無効も表示.value);

const handleReload = () => {
  userListTableRef.value?.loadData();
};
</script>

<template>
  <!-- ツールバー内のチェックボックス -->
  <div class="table-options">
    <label class="checkbox-label">
      <input type="checkbox" v-model="件数制限" @change="handleReload" />
      件数制限
    </label>
    <label class="checkbox-label">
      <input type="checkbox" v-model="無効も表示" @change="handleReload" />
      無効も表示
    </label>
  </div>

  <!-- 一覧テーブルコンポーネントへの props 渡し -->
  <UserListTable
    ref="userListTableRef"
    :件数制限="件数制限"
    :無効も表示="無効も表示"
    :有効列表示="有効列表示"
    :戻URL="編集戻URL"
  />
</template>

スタイル（.table-options）

チェックボックス群はツールバーの下、テーブルの直上に配置します。

.table-options {
  display: flex;
  align-items: center;

  margin-bottom: 8px;
}

.checkbox-label {
  display: flex;
  align-items: center;
  gap: 5px;
  font-size: 14px;
  cursor: pointer;
  user-select: none;
}

.checkbox-label input[type="checkbox"] {
  width: 16px;
  height: 16px;
  cursor: pointer;
  margin: 0;
}

チェックボックス変更時の再取得フロー

  1. チェックボックスが変更される（@change="handleReload"）
  2. 親コンポーネントが子の loadData() を呼び出す
  3. 子テーブルが最新の props（件数制限/無効も表示）でAPIへリクエスト
  4. 結果を再描画（無効も表示=ON なら「有効」列も自動表示）
