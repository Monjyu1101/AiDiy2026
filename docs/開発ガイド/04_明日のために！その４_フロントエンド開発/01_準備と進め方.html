01_準備と進め方

フロントエンド実装では、一覧と編集の責務を分離して組み立てると保守しやすくなります。ここで全体像を先に固めます。

今回作る画面

画面役割主な機能
M商品一覧検索・一覧表示の入口再検索、新規遷移、編集遷移
M商品編集新規/更新/削除入力検証、API呼び出し、一覧へ戻る

コンポーネント構成（推奨）

workspace/AiDiy2026/frontend_server/src/components/Mマスタ/M商品/
├── M商品一覧.vue
├── M商品編集.vue
└── components/
    └── M商品一覧テーブル.vue

実装の順序

  1. 編集画面（M商品編集.vue）を作る
  2. ルーティングを設定する
  3. 一覧テーブル（M商品一覧テーブル.vue）を作る
  4. 一覧画面（M商品一覧.vue）を作る
  5. メニュー導線を追加する
  6. E2Eで登録→編集→削除を確認する

API接続の前提

  • 一覧: POST /apps/V商品/一覧
  • 取得: POST /apps/M商品/取得
  • 登録: POST /apps/M商品/登録
  • 変更: POST /apps/M商品/変更
  • 削除: POST /apps/M商品/削除
  • 呼び出し実装: workspace/AiDiy2026/frontend_server/src/api/client.ts の apiClient

事前チェック

  • [ ] http://localhost:8090 にアクセスできる
  • [ ] http://localhost:8092/docs でM商品APIを確認できる
  • [ ] フロントの基底APIクライアント（apiClient）が利用できる
  • [ ] 実装は script setup + TypeScript を前提にする
  • [ ] 日本語コンポーネントタグを直接書かない（必要時は <component :is="..." />）
