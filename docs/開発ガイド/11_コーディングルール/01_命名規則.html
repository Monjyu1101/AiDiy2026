テーブル命名規則

全テーブル名はプレフィックス1文字＋日本語業務名。Database VIEW オブジェクトは作成しない（V系は生 SQL で実装）。

プレフィックス種別例主なサーバー
CCore / CommonC権限, C利用者, C採番core_main (8091)
MMasterM車両, M商品, M商品分類, M配車区分, M生産区分, M生産工程, M商品構成apps_main (8092)
TTransactionT配車, T商品入庫, T商品出庫, T商品棚卸, T生産apps_main (8092)
VView（生 SQL）V利用者, V商品, V配車, V生産, V商品推移表core / apps
SScheduler / SpecialS配車_週表示, S配車_日表示, S生産_週表示, S生産_日表示apps_main (8092)

API Endpoint 命名規則

全エンドポイントは POST メソッド。動詞は日本語統一。サーバーに応じて /core/ または /apps/ プレフィックスを使う。

動詞用途例（C利用者）例（M商品）
一覧一覧取得POST /core/C利用者/一覧POST /apps/M商品/一覧
取得1件取得POST /core/C利用者/取得POST /apps/M商品/取得
登録新規作成POST /core/C利用者/登録POST /apps/M商品/登録
変更更新POST /core/C利用者/変更POST /apps/M商品/変更
削除論理削除（有効=False）POST /core/C利用者/削除POST /apps/M商品/削除
 
# 正しいパターン（全て @router.post）
router = APIRouter(prefix="/apps/M商品", tags=["M商品"])

@router.post("/一覧", response_model=schemas.ResponseBase)
def list_M商品(db: Session = Depends(deps.get_db), ...): ...

@router.post("/取得", response_model=schemas.ResponseBase)
def get_M商品(request: schemas.M商品Get, db: Session = Depends(deps.get_db), ...): ...

@router.post("/登録", response_model=schemas.ResponseBase)
def create_M商品(request: schemas.M商品Create, db: Session = Depends(deps.get_db), ...): ...

@router.post("/変更", response_model=schemas.ResponseBase)
def update_M商品(request: schemas.M商品Update, db: Session = Depends(deps.get_db), ...): ...

@router.post("/削除", response_model=schemas.ResponseBase)
def delete_M商品(request: schemas.M商品Delete, db: Session = Depends(deps.get_db), ...): ...


# 誤り（GET / PUT / DELETE は使用禁止）
@router.get("/一覧")     # NG
@router.put("/変更")     # NG
@router.delete("/削除")  # NG

日本語変数名の使い方

業務ドメイン用語は日本語、フレームワーク・ライブラリ用語は英語。

種別言語例
業務エンティティ（変数・引数）日本語利用者ID, 配車日付, 商品名, 有効
業務関数名日本語利用者一覧取得(), 商品登録処理()
業務クラス（ORM モデル）日本語class C利用者, class M商品
フレームワーク変数英語router, db, request, response
一覧 API 共通キー英語items, total, limit, query
Vue / Pinia英語ref(), computed(), useRouter(), loading
定数英語 UPPER_SNAKEMAX_ITEMS = 10000, LIMITED_MAX_ITEMS = 1000
 
# Python バックエンド
MAX_ITEMS = 10000                           # 定数: 英語 UPPER_SNAKE

Vue コンポーネント命名

ファイル名は日本語でよい。ただし HTML テンプレート内のタグ名は ASCII のみ有効。日本語コンポーネントは <component :is="..."> で使う。

// ファイル名: C利用者一覧.vue （日本語OK）

// router/coreRouter.ts
{
  path: '/C管理/C利用者/一覧',
  name: 'C利用者一覧',
  component: () => import('@/components/C管理/C利用者/C利用者一覧.vue')
}
<!-- 誤り: 日本語タグ名は HTML として無効 -->
<C利用者一覧 />

<!-- 正解: 動的コンポーネント構文を使う -->
<component :is="C利用者一覧コンポーネント" />

CRUD 画面のファイル構成パターン:

src/components/M管理/M商品/
  M商品一覧.vue            ← ルートコンポーネント（親・件数制限/無効も表示の state を持つ）
  M商品編集.vue            ← 編集ダイアログ
  components/
    M商品一覧テーブル.vue  ← 一覧テーブル（子・props で件数制限/無効も表示を受け取る）

JSON キー命名

リクエスト・レスポンス JSON のキーも日本語が原則。以下の共通項目は英語のまま使用する。

英語キー（固定）意味使用場所
status"OK" / "NG"全レスポンス
messageメッセージ文字列全レスポンス
dataレスポンスデータ本体全レスポンス
items一覧配列一覧レスポンス
total総件数一覧レスポンス
limit取得上限数一覧レスポンス
errorエラー詳細オブジェクトNG レスポンス
 
// 一覧レスポンスの例
{
  "status": "OK",
  "message": "商品一覧を取得しました",
  "data": {
    "items": [
      { "商品ID": "G001", "商品名": "サンプル商品", "有効": true }
    ],
    "total": 42,
    "limit": 10000
  }
}

// リクエストの例（業務フィールドは日本語）
{
  "商品ID": "G001",
  "件数制限": true,
  "無効も表示": false
}
