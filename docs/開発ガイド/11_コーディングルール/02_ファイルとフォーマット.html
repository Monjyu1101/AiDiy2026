エンコーディング

全ファイルは UTF-8（BOM なし） 必須。Python ファイルは先頭行に # -*- coding: utf-8 -*- を付ける。

# -*- coding: utf-8 -*-
# ↑ 全 Python ファイルの先頭に必須

VSCode 設定（.vscode/settings.json またはユーザー設定）:

{
  "files.encoding": "utf8",
  "files.autoGuessEncoding": false
}

確認方法（Windows PowerShell）:

Get-Content -Path ファイル名.py -Encoding UTF8
  • BOM あり UTF-8（UTF-8 with BOM）は不可。ファイル保存時に必ず確認する
  • Windows メモ帳はデフォルトで BOM を付けることがあるため使用しない

コメント

コメントは業務ロジックに関する補足のみ書く。コードを読めばわかる自明な処理には不要。

# 正解: 業務的な理由が必要な箇所にのみコメント
item.有効 = False   # 論理削除: 物理削除は行わない
db.refresh(item)    # フロント側の状態更新のためリフレッシュが必要

# 誤り: コードそのままの説明は不要
item.有効 = False   # 有効を False にする  ← 不要
db.commit()         # コミットする          ← 不要
// 正解: 業務的な補足にのみコメント
// 論理削除済みレコードは一覧に表示しない（デフォルト動作）
const 無効も表示 = ref(false)

// 誤り: Vue の文法そのままの説明は不要
const loading = ref(false)  // ローディング状態を管理する ref  ← 不要

重要ファイルの構成

以下のファイルはプロジェクト全体で共有される。新規エンドポイントや画面を追加するときは必ず参照・活用すること。

バックエンド共通ファイル

ファイルパス役割

M系・C系（MAX_ITEMS 固定）と V系・T系（list_controls）の使い分け

# M系・C系: リクエストボディなし、MAX_ITEMS 固定
MAX_ITEMS = 10000  # Router ファイル上部に定義

items = (
    db.query(models.M商品)
    .filter(models.M商品.有効 == True)
    .order_by(models.M商品.商品ID)
    .limit(MAX_ITEMS)
    .all()
)
return {"items": items, "total": total, "limit": MAX_ITEMS}


# V系・T系: ListRequest を受け取り list_controls を使う
from list_controls import append_active_condition, get_limit_clause

conditions = []
append_active_condition(conditions, request, "M.有効")
limit_sql, limit_value = get_limit_clause(request)
params = {}
if limit_value is not None:
    params["limit"] = limit_value

sql = f"""
    SELECT ...
    FROM M商品 M
    WHERE {' AND '.join(conditions)}
    ORDER BY M.商品ID
    {limit_sql}
"""
result = db.execute(text(sql), params).fetchall()
