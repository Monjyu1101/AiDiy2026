D:/OneDrive/_sandbox/AiDiy2026/AGENTS.md, D:/OneDrive/_sandbox/AiDiy2026/CLAUDE.md 【Claude Code の要点】Anthropic の agentic coding system。コードベースを読み、複数ファイルを変更し、テストを実行し、コミット可能なコードを届けるプロジェクトレベルのエージェントとして説明されている。公式 docs では terminal で動作し、機能構築、デバッグ、コードベース理解、MCP 経由の外部データソース連携、CI やリリースノートなどの自動化に触れている。強みは terminal / Unix 的ワークフロー、既存ツールチェーンとの親和性、コードベース全体の文脈把握。注意点はファイル変更やコマンド実行の許可管理、人間によるレビュー、プロジェクト固有ルールの明示が重要であること。 【OpenAI Codex の要点】OpenAI の software development 向け coding agent。Codex manual ではコード作成、未知のコードベース理解、コードレビュー、デバッグ、開発タスク自動化に使えると説明されている。Codex CLI は terminal 上の lightweight coding agent で、ローカルマシン上でコードを読み、変更し、コマンドを実行できる。Codex は CLI / IDE extension / web / desktop app / cloud など複数 surface を持ち、Codex app は複数 agents を並列に管理する command center として説明されている。MCP は CLI と IDE extension でサポートされ、Codex に外部ドキュメントや開発ツールへのアクセスを与えられる。強みはローカル terminal とクラウド/アプリの両方を持つ多面展開、sandbox/approval/config による制御、MCP 連携。注意点は surface ごとの役割差を混同しないこと。 【Google Antigravity の要点】Google の agentic development platform。公式 docs では agent-first era に向けた開発プラットフォームで、Agent Manager と AI-powered IDE を持ち、agents を editor / terminal / browser にまたがって自律的に動かし、tasks と artifacts で高レベルに伝えると説明されている。公式 blog では browser control、asynchronous interaction、agent-first product form factor、task list / implementation plan / screenshots / browser recordings などの artifacts、ローカル操作からのフィードバック、knowledge base による self-improvement が強調されている。強みは IDE + browser + terminal をまたいだ視覚的検証と複数 agent 管理。注意点は agent 自律性が高いほど権限境界、成果物確認、破壊的操作の制御が重要であること。 【比較軸】1. 主な作業面: Claude Code は terminal 中心、Codex は CLI/IDE/app/cloud の複数 surface、Antigravity は agent-first IDE + Manager。2. 実行スタイル: Claude Code は開発者の既存 terminal とツールチェーンに寄り添う、Codex はローカル作業から並列 agent 管理まで広げる、Antigravity は editor/terminal/browser を横断する視覚的・非同期 agent 管理を狙う。3. 検証: Claude Code は tests/CI/commands、Codex は diffs/review/sandbox/MCP、Antigravity は artifacts/screenshots/browser recordings と task-level verification。4. 連携: Claude Code と Codex は MCP 連携を明示、Antigravity は artifacts/knowledge/manager を中心に説明する。5. 選び方: terminal で深く回すなら Claude Code、OpenAI 系モデルと CLI/IDE/app/cloud を横断したいなら Codex、ブラウザ検証や複数 agent を視覚的に監督したいなら Antigravity。 【AiDiy の立ち位置】AiDiy は Claude Code / Codex / Antigravity と競合する単一の coding agent ではなく、日本語ファーストの業務システム開発テンプレート兼、複数 AI / Code CLI / MCP / Avatar を束ねる実験基盤として位置づける。現行 AGENTS.md / CLAUDE.md に基づき、AiDiy は FastAPI + SQLite + Vue 3、frontend_avatar、command_hermes、15 MCP、AI コア、Code AI 設定（claude_sdk, claude_cli, copilot_cli, codex_cli, antigravity_cli, opencode_cli, aidiy_hermes）を持つ。つまり AiDiy は『どの agent を使うか』を選ぶ場所ではなく、『複数 agent を業務システム、Web UI、音声、画像、MCP とつなげて試す土台』として説明する。 【表現方針】勝敗ランキングにしない。『Claude Code は現場 terminal に強い』『Codex は OpenAI の複数 surface と MCP 連携に強い』『Antigravity は agent-first IDE と browser 検証に強い』『AiDiy はそれらを日本語業務システム開発の実験基盤へ接続する』という整理にする。事実は公式情報の範囲に限定し、未確認の価格、最新モデル名、性能順位、利用制限、内部実装は創作しない。誇大表現（完全自動開発、人間不要、絶対安全、どれか一つが完全上位など）は forbidden_elements に入れる。 【シーン構成】最小構成 7 シーンで生成すること。scene_000（イントロ: 3つの coding agent と AiDiy の立ち位置を比較する動画。この動画は AiDiy の video_generation 機能で自動生成された旨を冒頭で明言）、scene_001（Claude Code: terminal / codebase / tests / existing toolchain）、scene_002（Codex: CLI / IDE / app / cloud / sandbox / MCP）、scene_003（Antigravity: agent-first IDE / Manager / browser control / artifacts）、scene_004（比較表: 作業面・検証・連携・向いている用途）、scene_005（AiDiy: 日本語業務テンプレート + 15 MCP + Avatar + Code AI の受け皿）、scene_999（まとめ: 選び方と AiDiy で試す誘導）。scene_006 以降は作成しない。 【scene_999 の締め必須3要素】(1) この動画は AiDiy の video_generation
