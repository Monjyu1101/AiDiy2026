🇯🇵 日本語ファースト — 実装詳細

DB から API、UI まで名前をズラさない。

🗄️ Database

テーブル名・カラム名まで日本語で統一。ビジネス用語とコードの乖離をゼロにします。

Table:  C利用者
Column: 利用者ID
Column: 登録日時
Column: 更新利用者名

⚡ API

エンドポイント・JSON キーも日本語。POST 中心の統一 API 設計で一貫性を保ちます。

POST /core/利用者/一覧
POST /apps/商品/登録

{ "利用者ID": "admin",
  "権限ID":   "1" }

🎨 Frontend

Vue ルート・コンポーネントファイル名も日本語。ファイル名は日本語 OK、タグは ASCII で動的参照。

Route: /C管理/C利用者/一覧
File:  C利用者一覧.vue
Key:   利用者名

7 系統で全機能を分類。

C系 — Core

権限・利用者・採番

M系 — Master

配車区分・車両・商品・工程

T系 — Transaction

配車・生産・入庫・出庫・棚卸

V系 — View

生 SQL クエリによる結合表示

S系 — Scheduler

週表示・日表示スケジューラー

A系 — AI

AI コア・会話履歴

X系 — Experimental

ゲーム・自己紹介ページ

🖥️ マルチサーバー構成 — 実装詳細

構成は分かれていても、体験はひとつに寄せる。

🗄️ backend_server

FastAPI + SQLAlchemy + SQLite。POST 中心 API、統一レスポンス形式、C採番、監査フィールド、Reboot 機構を標準装備。Alembic 不使用。

core_main.py:8091apps_main.py:8092No Alembic

🔌 backend_mcp

FastAPI + SSE で 6 つの MCP サーバーを同一ポートに同居。AI エージェントのブラウザ操作・画面キャプチャ・DB 観測・ログ監視・型チェックを一手に提供。詳細は下の MCP Hub セクションへ。

mcp_main.py:8095SSE × 6Self-Verify

🌐 frontend_web

Vue 3 + Vite + TypeScript + Pinia。日本語ルーティング、qTubler カスタムテーブル、共通ダイアログ。CRUD 画面パターンで実装を統一。

Vue RouterPiniaqTubler

📱 frontend_avatar

Electron と通常ブラウザの両対応。BroadcastChannel で状態同期し、AI コアをマルチパネルで扱います。window.desktopApi でモード判定。

ElectronWeb ModeBroadcastChannel

🔒 認証 + 採番

JWT 認証（HS256・60分）を Pinia + Axios interceptor で自動管理。C採番テーブルで AUTOINCREMENT 不使用のカスタム ID を一元管理。

JWT / HS256C採番監査フィールド

共通基盤機能。

📊 qTubler

独自カスタムテーブル。ソート・ページング・行選択

🌐 WebSocket

AI コアリアルタイム通信。セッション永続化対応

📝 監査フィールド

全テーブルに登録 / 更新情報を自動付与

🚀 POST 中心 API

全 CRUD を POST で統一。一貫したレスポンス形式

🔄 Reboot 機構

txt ファイル作成でサーバー内部再起動

🎯 V系実装

Database VIEW 不使用。生 SQL クエリで JOIN

🧠 自己改善機構 — 実装詳細

使うほど育つ .aidiy/ 知見システム。

🧠

.aidiy/ 知見フォルダ

コードエージェントは修正が完了するたびに、
プロジェクト専用の .aidiy/ フォルダへ
知見を自動整理します。

次回の同種修正では .aidiy/_index.md を参照し、
過去の成功パターンを再利用して精度を高めます。
使うほどに、システム自体が育ちます。
📚

運用フロー

修正前に _index.md と
_最終変更.md を参照。
修正完了後にテーマ別メモへ追記・索引更新。

単なる作業ログではなく
「次回の修正判断に使える情報」を残します。
仕様はソースへ、知見は .aidiy へ。

🔌 MCP Hub × 6 + Multi CLI — 実装詳細

ポート 8095 に 6 つの MCP サーバーを同居。人の手数を減らし、AI の自己検証を増やす。

🌐 aidiy_chrome_devtools

Chrome DevTools Protocol を Python 純正実装で MCP 化。navigate / click / screenshot / DOM 取得・JS 実行まで。

CDPSSEPython

📸 aidiy_desktop_capture

全画面・領域指定・ウィンドウ指定・マルチモニター対応のスクリーンショット取得。AI が UI を目視確認するチャネル。

pyautoguiPILmulti-monitor

🗄️ aidiy_sqlite

AiDiy DB に対し、スキーマ確認 / 件数 / 監査フィールド抽出 / SELECT を安全に実行。既定は read-only、書き込みは明示許可のときのみ。

read-only defaultPRAGMAaudit summary

🐘 aidiy_postgres

外部 PostgreSQL に psycopg 3 で接続。DB/スキーマ/テーブル探索、describe、件数、任意 SELECT。接続先は環境変数 or 呼び出し時の DSN 引数。

psycopg 3read-only txstatement_timeout 15s

📜 aidiy_logs

backend_server と backend_mcp のログを tail、正規表現 grep、ERROR/Traceback 抽出（前後 2 行の文脈付き）。変更直後の回帰検知に。

tailgreperror context

✅ aidiy_code_check

Python 構文（py_compile）・ruff lint・TypeScript 型チェック（npm run type-check）を subprocess で実行。副作用なしの検査系だけ。

py_compilerufftsc
🔁

Self-Verification Loop

「コードを直す → DB で結果を見る →
ログでエラーを見る → 型で再確認」を AI が自走。
人間は指示と判断に集中できます。

edit → observe → verify
MCP 6 サーバーの連携で、修正の影響範囲を
AI が自分で確認してから報告します。
