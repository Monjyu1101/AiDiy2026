# AiDiy 自己紹介動画シナリオ (take3 / 5ページ版)

入力元: `frontend_web/public/X自己紹介/index.html`

想定尺: 約60秒
形式: 音声付き HTML → MP4 変換前提

## 全体方針

- コンセプトは「AI × DIY 感覚でシステムづくり。でも中身は本気。」
- 楽しさを前面に、裏にある本格設計を見せる二層構造。
- 5 シーンに圧縮。ナレーション約 45 秒 + 映像余白で 60 秒以内。
- トーン: 軽快で親しみやすく、でも芯は熱い。
- scene_003（アバター）と scene_004（マルチエージェント）を「AI パートナー」として統合。

## シーン構成

### scene_001: オープニング

表示:

```text
AiDiy
AI × DIY 感覚で、自分の手でシステムを作る。
```

ナレーション:

> AiDiy。AI と一緒に、DIY 感覚で業務システムを作る。でも中身は本気です。

狙い:

- 名前とコンセプトを 5 秒で固定。「楽しい」と「本気」の両方を最初に宣言。

### scene_002: 日本語 × 本格設計

表示:

```text
日本語で書ける。中身は 4 層アーキテクチャ。
テーブル監査証跡も完備。
```

ナレーション:

> データベースも API も画面も、日本語でそのまま。難しい言葉はいりません。でも中身は Model / Schema / CRUD / Router の 4 層構造。全テーブルに監査証跡も入った、本格設計です。

狙い:

- 日本語ファーストの取っつきやすさと、エンタープライズ級の設計を同時に伝える。

### scene_003: ライブ会話 × アバター

表示:

```text
話しかければ、応えてくれる。
VRM アバター × 音声 × AI
```

ナレーション:

> コードを書くだけじゃありません。話しかければ AI が応える。アバターがうなずき、あなたの相棒になる。チャットも音声も、ライブでつながります。

狙い:

- 「使う楽しさ」をアバターと音声会話で見せる。見た目のインパクトも大きい。

### scene_004: 本格マルチエージェント

表示:

```text
Claude / Copilot / Gemini / Codex / OpenCode
あなたの最適な相棒を選べる
```

ナレーション:

> コード支援はひとつじゃない。Claude、Copilot、Gemini、Codex、OpenCode。複数の AI エージェントを用途で使い分け。最適な相棒を選んで、一緒に作れます。

狙い:

- マルチエージェントの実用性。単なる AI 頼りではなく、道具を使いこなす姿勢。

### scene_005: 実用 MCP 道具箱

表示:

```text
MCP Hub × 13
ブラウザ操作 / 画面キャプチャ / 画像生成
音声合成 / DB観測 / 動画編集 / バックアップ …
観測・生成・検証。全部そろってる。
```

ナレーション:

> AI の手足になる 13 の MCP ツール。ブラウザ操作に画面キャプチャ、画像生成、音声合成、動画編集、バックアップ。観測、生成、検証。全部そろってるから、AI の届く範囲が違います。

狙い:

- MCP の数と具体性で「実用性」を見せる。遊びじゃない道具群。

### scene_006: はじめよう

表示:

```text
AiDiy
動くサンプル完備。一発セットアップ。
使うほど育つ基盤。
```

ナレーション:

> 配車、生産、在庫。動くサンプルは揃ってます。セットアップは一発。使うほど知見が溜まって、あなただけの基盤に育ちます。AiDiy、はじめよう。

狙い:

- すぐ始められる + 継続的価値でクロージング。「やってみたい」で終わる。
