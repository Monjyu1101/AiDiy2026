- `backend_mcp` の入口を `mcp_main.py`、再利用ロジックを `mcp_proc/` へ分離する構成に整理した
- `_start.py` の `(mcp)` ブラウザ起動ヘルパーに残っていた旧 import を `mcp_proc.chrome_manager` へ揃えた
- `scripts/cli_bat/_codex_cli_AiDiy.bat` の `-c` 上書きを外し、引用符崩れを避ける形に寄せた
- Windows で `codex.cmd -c "mcp_servers...='http://...'"` が引用符崩れで起動失敗するケースを知見化した
- Codex の `url = ...` は streamable HTTP 用で、`backend_mcp` の SSE エンドポイントは直接扱えないと確認した
- `_setup.py` の `backend_mcp` 用 MCP 設定書き込みは Claude / Gemini のみに寄せ、Codex の `backend_mcp` 設定は自動削除するようにした

- `backend_mcp` の SSE 初回 `endpoint` イベントを、引用符付き文字列ではなく生の相対パスへ修正し、`/messages` と `/messages/` の両方を受けるようにした

- `backend_mcp/mcp_stdio.py` を追加し、Codex から stdio で起動して既存の `backend_mcp` SSE へ接続するブリッジを実装した
- `_setup.py` を修正し、Codex 用の `~/.codex/config.toml` へ `mcp_stdio.py --sse-url ...` の stdio bridge 設定を書き込むようにした
