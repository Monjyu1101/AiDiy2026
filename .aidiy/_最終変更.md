# 最終変更

更新日: 2026-04-19

## 今回整理した内容
- `frontend_web/public/X自己紹介/index.html` で VRMA 連続再生が消えていたため再実装
  - 症状: VRMA 再生が止まると T-pose に戻り、カメラが `-Z` 側から見ているため **背中を向けて両手を広げる** 姿に見える（「たたずんでる感じ」が失われる）
  - 対策1: `container.rotation.y = Math.PI` を入れ、VRM 正面（+Z）をカメラ側へ向ける。T-pose になっても少なくとも正面を向く
  - 対策2: `AnimationMixer` + `@pixiv/three-vrm-animation` の `VRMAnimationLoaderPlugin` / `createVRMAnimationClip` で VRMA を連続再生し、切れ目を作らない
  - 実装: `LoopOnce` + `clampWhenFinished = true` で各クリップを再生し、`mixer.addEventListener('finished', ...)` で次クリップへ接続。`crossFadeFrom(previousAction, 0.5, false)` で補間、初回のみ `fadeIn(0.5)` で T-pose から自然に入る
  - キュー: `VRMA_FILES` をシャッフルして巡回。先頭が前回再生と同じなら入れ替えて連続回避
  - 参照モデル: `frontend_avatar/src/components/AIコア_アバター.vue`（読み取り専用・直接触らない）

## 現時点の主要知見
- `アバター表示調整とVRMA再生の連続化.md`
  - Vue コンポーネント（AIコア_アバター.vue）ではカメラ距離倍率で制御
  - 静的HTML（X自己紹介/index.html）では CSS コンテナ寸法と `renderer.setSize(..., false)` の同期で制御
  - 静的HTML側で VRMA を途切れさせると T-pose に戻るので、**連続再生は必須**
  - VRM 正面は既定で +Z。カメラを -Z に置く構成なら `container.rotation.y = Math.PI` を合わせて入れる
  - 連続再生は `LoopOnce` + `finished` イベントで次クリップを `clipAction` → `crossFadeFrom` が扱いやすい（setTimeout 併用しなくても OK）
