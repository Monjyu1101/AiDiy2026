# クリーンアップ手順

`_cleanup.py` を変更するときの注意点をまとめる。

## 対象

- ルート `backup/`
- `backend_server/temp/`
- `backend_mcp/temp/`
- `backend_hermes/temp/`
- 各 `.venv` / `venv`、`node_modules`、`dist`、`__pycache__`

## 実装方針

- `temp` は中身だけ消さず、フォルダごと `remove_directory()` で削除する。
  - 起動時に `core_main.py` / `apps_main.py` / `mcp_main.py` などが必要な `temp` を再作成する。
  - `backend_hermes/temp` も CLI ツールの一時出力先なので、引き渡し前の整理対象に含める。
- ルート `backup/` は `_cleanup.py` の選択項目に含め、存在する場合は既定 `y` で削除確認する。
  - `docs/.../code_samples/_cleanup.py` には古いサンプルが残りやすいため、実装を変えたらサンプル側の文言も確認する。
- `logs` は現状どおり中身削除でよい。ログフォルダ自体を消す方針に変える場合は、ログ設定側の再作成処理を先に確認する。

## 検証

- `_cleanup.py` は `python -m py_compile _cleanup.py` で構文確認する。
- 削除挙動は一時ディレクトリ配下に `backend_server/temp`、`backend_mcp/temp`、`backend_hermes/temp`、`backup` を作って、`cleanup_backend()` / `cleanup_backend_mcp()` / `cleanup_backend_hermes()` / `cleanup_backup()` を直接呼ぶと安全に確認できる。
