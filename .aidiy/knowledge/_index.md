# .aidiy/knowledge インデックス

このフォルダは、AiDiy2026 専用の修正知見を蓄積する場所です。
アプリ本体の仕様や実装は通常のソースコードへ反映し、`.aidiy/knowledge` には次回の修正で再利用できる判断基準と注意点だけを残します。

## 参照順序
1. まずこの `_index.md` を確認する
2. 直近の流れを知りたい場合は `_最終変更.md` を確認する
3. 似た修正テーマがあれば、対応するテーマ別メモを確認する
4. 修正完了後は、必要な知見だけを既存メモへ統合する

## 現在の知見一覧

### `_最終変更.md`
- 直近で実施した修正・整理内容の要約
- 再修正依頼が来たときの最優先参照先

### `C利用者パスワード運用.md`
- C利用者のパスワード保存方式と認証互換の運用ルール
- 初期データ、通常登録、変更、認証の扱いを確認したいときに参照する

### `新たなCLI追加時の作業手順.md`
- 新しい code CLI を AiDiy に追加するときの標準チェックリスト
- backend 実装、設定JSON、設定UIを一式で揃える必要があるときに参照する

### `CLIプロンプト正規化と履歴保持.md`
- Code CLI の system prompt と送信テキストの整形責務を分けるときに参照する
- 履歴可読性を残しつつ、CLI 引数へ渡す直前だけ 1 行化したいときの判断基準をまとめたメモ

### `Code CLIのMCP設定.md`
- `claude` / `gemini` / `codex` に同じ MCP を使わせたいときに参照する
- CLI ごとの設定場所の違いを確認できる

### `backend_mcp構成整理.md`
- `backend_mcp` の入口とロジック配置を整理するときに参照する
- `mcp_main.py` と `mcp_proc/` の役割分担を確認できる

### `アバター表示調整とVRMA再生の連続化.md`
- アバターの表示サイズ、向き、VRMA のループ（連続）再生を調整するときに参照する
- Vue コンポーネント（AIコア_アバター.vue）と静的HTMLページ（X自己紹介/index.html）の両方を対象に、カメラ距離や初期回転、アニメーション設定をまとめたメモ

### `アバターAIファイル認証延長.md`
- AIファイル画面表示中、音声以外のAI入力系送信時、S/V更新日監視時、C/M/T操作系API送信時にJWTを延長するときに参照する
- Avatar側はPinia認証ストアではなく、Electron/Web別の認証Storageへ直接トークンを書き戻す
- Web側のAI入力系は `useAuthStore().refreshToken()` を使う

### `M系マスタ追加チェック.md`
- M商品分類 / M商品 と同じ型の M系マスタを追加するときに参照する
- backend の Model / Schema / CRUD / Router / V系Router / init 登録と、frontend の型・ルート・メニュー・一覧/編集画面を一式で揃えるためのチェック項目

### `Markdown現状追従チェック.md`
- `.md` 全体を現行実装へ追従させるときに参照する
- MCP 8サーバー構成、M取引先系、認証延長ルール、AI名キーなど、ズレやすい確認観点をまとめたメモ

### `X系静的画面追加.md`
- Xその他配下にゲーム・デモ・マスコットなどの静的画面を追加するときに参照する
- Vue iframe ラッパー、`public/<画面名>/`、`router/index.ts`、`Xその他.vue` の登録点を確認できる
- `react-neko` / `neko-ts` 系のような外部Nekoライブラリを扱う場合のライセンス注意点も記録

## 更新ルール
- ファイル名は内容が分かる日本語名を優先する
- 同じテーマの知見は新規ファイル乱立ではなく既存メモへ統合する
- 単なる作業ログではなく、次回の修正判断に使える再利用情報を残す
- 他プロジェクトの知見は混在させない
- 類似知見がない新規テーマだけを新しいメモとして追加する

## 追記時の最低記載項目
- 何の場面で参照する知見か
- 関連ファイルと関連箇所
- 実装や運用の結論
- 再発しやすい注意点
- 最低限の確認方法
