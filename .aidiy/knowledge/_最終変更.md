# 最終変更

更新日: 2026-04-24

## 今回整理した内容

- `frontend_web` に `X立体リバーシ` を追加した
  - `frontend_web/public/X立体リバーシ/` に 6面立方体リバーシ本体（HTML/CSS/JS）を新規作成した
  - `frontend_web/src/components/Xその他/X立体リバーシ.vue` を iframe ラッパーとして追加した
  - `frontend_web/src/router/index.ts` に `/Xその他/X立体リバーシ/ゲーム` を追加し、`Xその他.vue` にメニューカードを追加した
  - 4x4 の6面盤、斜めは同一面のみ、縦横は立方体面またぎ、チャレンジモード、CPU対戦、合法手表示、ルール明記を実装した
  - Three.js CDN ES module による 3D キューブ表示、ドラッグ自由回転、3D立方体上のクリック着手、着手後の低速反転アニメーション、自動カメラワークを追加した
  - 6面ネット表示は外し、3Dキューブ上で直接プレイする構成にした
- `frontend_avatar` の `表示選択` に `xeyes(目)` を追加した
  - `AIコア_xeyes.vue` を新規作成し、Web では画面内 pointermove、Electron では OS カーソル位置を使って目線を制御する
  - Electron main/preload/env に `getWindowPointerSnapshot(role)` を追加し、自分のウィンドウ role を渡して `bounds` と `mouse` を取得できるようにした
  - Electron main/preload/env に `getSystemCpuUsage()` を追加し、`AIコア_xeyes.vue` で 1 秒ごとの直近 10 秒 CPU 平均に応じて血走り演出を行うようにした
  - ボタン表示中だけ、xeyes の目の下に直近 10 秒 CPU 使用率グラフと平均値を表示するようにした
- `AIコア_デジタル時計.vue` の 7 セグメント参照を安全参照にして、型チェックが通るようにした
- `frontend_avatar` の `AIコア_ネコ.vue` を `AIコア_xneko.vue` へリネームした
  - `AIコア.vue` の import も新ファイル名へ更新した
  - `AIコア_xneko.vue` のスプライト参照に undefined ガードを追加し、型チェックを通した
- `AIコア.vue` の実験的アバター設定 UI を調整した
  - `アバター表示` チェックボックスを `表示選択` select に変更し、`アバター` / `xneko(猫)` / `無し` の 3 択にした
  - `不完全な自立身体制御` / `不完全な自動カメラワーク` / カメラモードは `AIコア_アバター.vue` 内へ移動した
  - `AIコア.vue` から `controls-visible` を渡し、ボタン非表示時はアバター内の設定 UI・字幕・発光を隠すようにした
- `表示選択` に `アナログ時計` / `デジタル時計` / `カレンダー` を追加した
  - 各表示は `AIコア_アナログ時計.vue` / `AIコア_デジタル時計.vue` / `AIコア_カレンダー.vue` の独立コンポーネントとして実装した
- `.aidiy/knowledge/アバター表示調整とVRMA再生の連続化.md` を更新し、表示選択と実験的設定 UI の配置を追記した

## 次回の運用

- `AIコア` のアバター固有設定は `AIコア_アバター.vue` 内へ置き、`xneko(猫)` / `無し` 選択時には表示しない
- `表示選択` を増やす場合は、独立コンポーネントを作って `AIコア.vue` の import / `表示選択型` / option / 表示 computed を揃える
- Electron でウィンドウ外カーソルを追う表示を作る場合は、renderer の mousemove ではなく main process の `screen.getCursorScreenPoint()` と `BrowserWindow.getBounds()` を返す IPC を使う
- Electron で CPU 使用率連動演出を作る場合は、main process の `os.cpus()` 差分を IPC で返し、renderer 側で用途に応じた移動平均を持つ
- ファイル名リネーム後は `rg` で旧名参照を確認し、`frontend_avatar` の型チェックまで実施する
- X系静的ゲームを増やす場合は、`public/<画面名>/` の本体、`src/components/Xその他/<画面名>.vue` の iframe ラッパー、`router/index.ts`、`Xその他.vue` の4点を揃える
