# 開発環境運用手順

> 文書: `共通,開発環境運用手順.md` | 実装: `_start.py`, `backend_server/_data/`, `frontend_web/`

## このメモを使う場面

- AiDiy2026 を起動、停止、再起動する。
- 依存関係を同期する。
- DB を初期化または確認する。
- Swagger や画面で最小動作確認を行う。
- ポート競合、CORS、401、database locked などを切り分ける。

## 起動手順

通常起動はプロジェクトルートで行う。

```powershell
python _start.py
```

`_start.py` は対話形式で、起動するサービスを選択する。`backend_hermes` は on-demand CLI のため常駐起動対象ではない。

個別起動が必要な場合:

```powershell
# Core API
Set-Location backend_server
.venv/Scripts/python.exe -m uvicorn core_main:app --reload --host 0.0.0.0 --port 8091

# Apps API
Set-Location backend_server
.venv/Scripts/python.exe -m uvicorn apps_main:app --reload --host 0.0.0.0 --port 8092

# MCP
Set-Location backend_mcp
.venv/Scripts/python.exe mcp_main.py

# Hermes CLI
Set-Location backend_hermes
.venv/Scripts/python.exe cli_main.py

# frontend_web
Set-Location frontend_web
npm run dev

# frontend_avatar
Set-Location frontend_avatar
npm run dev
```

## バックエンド変更の反映

`_start.py` 起動では uvicorn に `--reload` が付かない。コード変更を反映する場合は、次のいずれかを使う。

```powershell
# core_main.py を再起動
New-Item -ItemType File -Force backend_server/temp/reboot_core.txt

# apps_main.py を再起動
New-Item -ItemType File -Force backend_server/temp/reboot_apps.txt
```

または個別起動で `--reload` を使う。

## 初期セットアップ

初回または環境リセット時:

```powershell
python _setup.py
```

主な対象:

- `backend_server` の Python 仮想環境と依存関係。
- `backend_mcp` の Python 仮想環境と依存関係。
- `backend_hermes` の Python 仮想環境と依存関係。
- `frontend_web` / `frontend_avatar` の `npm install`。

## 依存関係

```powershell
# backend_server
Set-Location backend_server
uv sync
uv add <package>

# backend_hermes
Set-Location backend_hermes
uv venv .venv
uv pip install -r requirements.txt

# frontend_web
Set-Location frontend_web
npm install
npm run type-check

# frontend_avatar
Set-Location frontend_avatar
npm install
npm run type-check
```

`npm run build` は `dist/` を生成するため、明示指示がある場合または Docker 反映が必要な場合に限定する。

## クリーンアップ

不要な仮想環境、`node_modules`、`dist`、キャッシュ、DB を整理する場合:

```powershell
python _cleanup.py
```

削除対象は対話形式で確認される。ルート `backup/` や DB を消す場合は復旧材料が不要か確認する。

## DB 管理

DB ファイル:

```text
backend_server/_data/AiDiy/database.db
```

開発用に全テーブルを作り直す場合:

1. `_start.py` や個別起動中のサーバーを停止する。
2. SQLite Browser / DBeaver など DB 接続ツールを閉じる。
3. `backend_server/_data/AiDiy/database.db` を削除する。
4. backend を再起動する。

初期データは、対象レコードが未存在の場合のみ投入される。既存DBでは自動更新されない。

スキーマ差分を既存DBへ反映する場合は、DB削除ではなく [`スキーマ変更手順.md`](./スキーマ変更手順.md) を使う。

## API 確認

Swagger:

- Core API: `http://localhost:8091/docs`
- Apps API: `http://localhost:8092/docs`

Swagger で認証付き API を確認する場合:

1. 画面でログインする。
2. Storage から JWT token を確認する。
3. Swagger の `Authorize` に token を入れる。
4. CRUD は POST 前提で確認する。

## ポート競合

`_start.py` は 8090 / 8091 / 8092 を起動前に解放し、Avatar 起動時は 8099 も対象にする。

手動確認:

```powershell
netstat -ano | findstr :8091
taskkill /PID <pid> /F
```

## よくある問題

| 症状 | 主な確認先 |
|------|------------|
| `address already in use` | ポート残留を `netstat` で確認する |
| 文字化け | ファイルが UTF-8 か、PowerShell で `Get-Content -Encoding UTF8` を使っているか確認する |
| 401 | JWT期限、Storage、`認証延長ルール.md` を確認する |
| CORS | `core_main.py` / `apps_main.py` の allowed origins と Vite port を確認する |
| Python `ModuleNotFoundError` | 対象 backend で `uv sync` または requirements install を行う |
| npm package 不足 | 対象 frontend で `npm install` を行う |
| `database is locked` | SQLite Browser / DBeaver などの接続を閉じる |
| テーブル不足 | backend 再起動、model import、`apps_models/__init__.py`、`apps_crud/__init__.py` を確認する |
| backend 変更が反映されない | `_start.py` 起動では `--reload` なし。reboot ファイルまたは個別起動を使う |
| M系一覧エラー | frontend は V系一覧を使うため、M系追加時は V系エンドポイントも作成する |

## 最小確認

- `http://localhost:8090` でログイン画面を開ける。
- Core Swagger `http://localhost:8091/docs` が開く。
- Apps Swagger `http://localhost:8092/docs` が開く。
- 既存の M / T / V 画面で一覧取得が通る。
- backend 変更後は `python -m py_compile`、router import、Swagger 表示のいずれかで確認する。
